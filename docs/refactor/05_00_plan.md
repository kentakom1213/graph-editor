# Phase 5 Refactor Plan: テスト追加

対象: docs/refactor/00_01_checklist.md の Phase 5 チェックリストを詳細化する。

## ゴール
- 重要ユースケース (parse / rebuild / 変換) をテストでカバーする。
- リファクタリング後の回帰を検出できる基盤を作る。

## 現状把握 (対象ファイル)
- `BaseGraph::parse` や `Graph::rebuild_from_basegraph` に単体テストがない。
- 描画や UI 操作は E2E テストが無いため、ロジック系を優先する。

## 方針 (決めるべきこと)
- ユースケース優先で「壊れると困るロジック」を先にテストする。
- UI 依存のテストは避け、純粋ロジックの検証に集中する。
- テストの粒度は「入力→出力の整合性」レベルで十分とする。

## 実施計画

### 1. 優先ユースケースの洗い出し
- parse:
  - 正常系 (最小グラフ / 複数辺)
  - 異常系 (フォーマット不正 / 範囲外 ID)
- rebuild:
  - 頂点数/辺数が一致すること
  - 無向/有向での辺の扱い
- 変換:
  - `calc_complement` / `calc_reverted` の整合性

### 2. テスト追加の設計
- `tests/graph_parse.rs` : `BaseGraph::parse` の入出力確認
- `tests/rebuild.rs` : `rebuild_from_basegraph` の頂点数/辺数確認
- `tests/transform.rs` : complement / reverted の変換確認

### 3. テスト実装
- 固定入力で期待値比較
- 失敗ケースは `anyhow::Result` を検証

### 4. 動作確認
- `cargo test` を実行し、全テストが通ることを確認する。

## 影響範囲と注意点
- `rebuild_from_basegraph` は乱数要素が含まれるため、座標の一致は検証しない。
- テストは構造的な一致 (頂点数/辺数) に限定する。

## 完了条件
- 優先ユースケースがテスト化されている。
- `cargo test` が通る。

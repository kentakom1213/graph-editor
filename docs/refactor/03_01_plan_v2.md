# Phase 3 Refactor Plan v2: 安定ID + Vecベースの操作

対象: wasm を考慮し、`Vec` のシーケンシャルアクセスを維持しつつ安定IDを導入する。

## ゴール
- 安定ID (`VertexId`) を導入し、削除や並び替えで参照が壊れないようにする。
- 描画・更新は `Vec` の走査を維持し、パフォーマンスを確保する。
- ID → index の解決は軽量な補助構造で行い、必要箇所だけ O(1) を実現する。

## 方針
- `VertexId` は新規型 (`struct VertexId(u64)` など) として導入する。
- `Vec<Vertex>` を維持し、`Vertex` 自体は `id: VertexId` を保持する。
- `Graph` に `id_index: HashMap<VertexId, usize>` のような補助マップを持ち、
  参照解決が必要な箇所だけ O(1) にする。
- `Edge` は `from/to: VertexId` を保持する。

## 実施計画

### 1. `VertexId` の導入
- `VertexId` 型を定義し、`Clone + Copy + Hash + Eq` を実装する。
- `Graph` に `next_vertex_id` を追加し、追加時に単調増加で発番する。
- `Vertex` の `id: usize` を `id: VertexId` に置き換える。

### 2. `Graph` の補助マップ導入
- `Graph` に `id_index: HashMap<VertexId, usize>` を追加する。
- 頂点追加・削除時に `id_index` を更新する。
- `apply_deletions()` で削除後に `id_index` を再構築する。

### 3. `Edge` の参照変更
- `Edge` の `from/to` を `VertexId` に変更する。
- `Graph::add_unique_edge()` は `VertexId` を使う形に修正する。

### 4. `GraphViewState` の移行
- `GraphViewState.vertices` は `Vec<VertexViewState>` を維持する。
- `GraphViewState` が `VertexId` による参照を必要とする場合は、
  `Graph` の `id_index` を利用する。
- `GraphSnapshot` 生成時は `Vec` を走査し、対応する view を index で参照する。

### 5. 描画/操作/エクスポートの更新
- 描画は `Vec` の走査を維持し、`VertexId` の解決が必要な箇所だけ `id_index` を使う。
- エクスポートも `Vec` 走査を前提にする。
- `.find()` での頂点探索は削除し、`id_index` に置き換える。

### 6. 動作確認
- 頂点追加/削除/辺追加/移動/色変更の一連動作を確認する。
- `cargo check` を通す。

## 影響範囲と注意点
- ID 再採番は行わないため、削除済み ID を参照しているケースに注意。
- `id_index` の更新漏れがあるとクラッシュや誤参照の原因になる。
- `Vec` の並び順を変える場合は `id_index` の再構築が必須。

## 完了条件
- `VertexId` が導入され、`Edge` が `VertexId` を保持している。
- 描画/操作が `Vec` ベースで動作し、必要な箇所のみ `id_index` を使っている。
- `cargo check` が通る。

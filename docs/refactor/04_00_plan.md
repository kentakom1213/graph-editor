# Phase 4 Refactor Plan: エクスポートと設定の整理

対象: docs/refactor/00_01_checklist.md の Phase 4 チェックリストを詳細化する。

## ゴール
- エクスポート処理を `GraphEditorApp` から分離し、責務と状態遷移を明確化する。
- `AppConfig` の拡張方法を整理し、UI での切り替えやシリアライズに耐える構造にする。

## 現状把握 (対象ファイル)
- エクスポート処理は `GraphEditorApp` に状態と処理が集中
  - `src/app.rs` (request/export events)
  - `src/export.rs` (export 実装)
- `AppConfig` が `Box<dyn Visualizer>` / `Box<dyn Simulator>` を保持
  - `src/config.rs`
  - UI での切り替えや保存に不向き

## 方針 (決めるべきこと)
- `ExportService` の責務範囲と状態遷移を定義する。
  - 例: `Idle -> Requested -> Capturing -> Writing -> Completed/Failed`
- `GraphEditorApp` からエクスポート処理を分離し、呼び出し点を単純化する。
- `AppConfig` を「列挙型 + パラメータ構造体」に整理する。
  - 例: `VisualizerKind` + `VisualizerConfig`
  - 例: `SimulatorKind` + `SimulatorConfig`

## 実施計画

### 1. `ExportService` の設計
- 役割と API を定義する。
  - `request_export(format)`
  - `handle_events(ctx, graph_view)`
  - `is_busy()`
- 失敗時の状態遷移とエラー保持を明確化する。
- `ExportState` を `ExportService` に集約するか、`ExportService` 内で管理するか決める。

### 2. `GraphEditorApp` からの分離
- `GraphEditorApp` からエクスポート関連のフィールドを削除/移動する。
- `components/edit_menu.rs` の export ボタンは `ExportService` を呼ぶだけにする。
- `export.rs` 側は `ExportService` から呼び出す形に整理する。

### 3. `AppConfig` の拡張方法整理
- `AppConfig` から `Box<dyn Visualizer>` / `Box<dyn Simulator>` を外す。
- `VisualizerKind` / `SimulatorKind` を追加し、UI での切り替えが可能な形にする。
- `VisualizerConfig` / `SimulatorConfig` を構造体で保持し、
  画面設定やシリアライズを想定できる形にする。

### 4. UI と保存形式の整理
- UI の選択が `Kind` を切り替える形で動作するように調整する。
- `UiState` の保存/復元に `VisualizerKind` / `SimulatorKind` を含めるか検討する。
- 既存の `AppConfig` との互換性を保つ方針を決める。

### 5. 動作確認
- エクスポートの一連動作が従来と同じ挙動になるか確認する。
- 設定切り替えが UI で動作し、保存/復元に破綻がないか確認する。
- `cargo check` を通す。

### 6. 最小限のファイル分割
- 1ファイルあたりの行数増加に備えて、最低限の分割を行う。
  - `src/app.rs`: `AppState` / `UiState` / `ExportState` を `src/state/` に移動する。
  - `src/export.rs`: `ExportService` 導入後に `service` と `codec` を分離する。
- 分割は 2〜3 ファイル程度に留め、依存関係を増やしすぎない。
- `mod.rs` を追加し、公開 API を固定する。

## 影響範囲と注意点
- `export.rs` の API 変更により呼び出し側の修正が広がる可能性がある。
- `AppConfig` の変更は UI/保存形式/初期化に影響する。
- wasm 環境では export 処理の制限があるため、`ExportService` で分岐整理が必要。

## 完了条件
- `ExportService` が導入され、`GraphEditorApp` は呼び出しのみを担当している。
- `AppConfig` が列挙型 + パラメータ構造体の形に整理されている。
- `cargo check` が通る。

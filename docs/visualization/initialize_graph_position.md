# グラフ初期配置の概要

`src/graph/visualizer.rs` では，グラフの頂点を 2 次元平面に初期配置するための仕組みと手法を定義している．

## Visualizer トレイト

- `resolve_vertex_position(n, edges) -> Vec<egui::Vec2>`
- 入力は頂点数 `n` と辺集合 `edges`（無向グラフを想定）．
- 出力は各頂点の座標で，範囲は `(0,1)^2` に収まることを想定している．

## 付随するユーティリティ

- `sample_point()`: `[0,1]^2` から一様ランダムに点を生成する．
- `cross(a, b)`: 2 次元ベクトルの外積（スカラー値）．
- `is_crossing((p1,q1),(p2,q2))`: 2 本の線分が厳密に交差するか判定する．
- `count_edge_crossing(positions, edges)`: 全辺ペアを走査して交差数を数える（計算量 `O(m^2)`）．

## 実装されている可視化手法

### Naive

- 各頂点を独立に一様乱数で配置するだけの単純な初期化．
- 交差数などの評価は行わない．

### HillClimbing(max_iter)

- 初期配置をランダムに作成した後，辺の交差数を評価指標として山登り法を行う．
- 反復ごとに 1 頂点をランダムに選び，その座標のみ再サンプリングする．
- 交差数が減った場合のみ更新し，`max_iter` 回繰り返す．
- `n == 0` の場合は空ベクトルを返す．

### SimulatedAnnealing { max_iter, initial_temp, cooling_rate }

- 初期配置をランダムに作成し，辺の交差数を評価指標として焼きなまし法を行う．
- 反復ごとに 1 頂点をランダムに選び，その座標のみ再サンプリングする．
- 交差数が減る遷移は常に受理し，増える遷移は温度 `T` に応じて確率的に受理する．
- 温度は `T <- T * cooling_rate` で減衰し，`max_iter` 回繰り返す．
- `n == 0` または `max_iter == 0` の場合はランダム配置を返す．
